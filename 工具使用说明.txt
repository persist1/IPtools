          GitHub 自动化助手 - 使用说明
====================================================================

🎉 超级简单！所有操作都是点击按钮完成！

====================================================================
🚀 快速开始
====================================================================

第 1 步：启动工具
-----------------
双击运行 "启动工具.bat"

如果提示 Python 未安装：
  访问 https://www.python.org/downloads/
  下载并安装 Python 3.7+


第 2 步：使用工具
-----------------
工具界面有 6 个按钮，按顺序点击：

1. 📤 一键上传到 GitHub
   - 自动配置 Git
   - 自动提交代码
   - 自动推送到 GitHub
   - 弹出密码输入框时输入密码

2. ✅ 启用 GitHub Actions
   - 自动打开浏览器
   - 点击绿色按钮启用

3. 🔨 触发自动构建
   - 自动触发第一次构建
   - 打开浏览器查看进度

4. 👀 查看构建状态
   - 随时查看构建进度
   - 下载构建完成的程序

5. 🏷️ 创建发布版本
   - 输入版本号（如 v1.0.0）
   - 自动创建正式发布
   - 自动构建并上传安装包

6. 🌐 打开 GitHub 仓库
   - 直接打开你的仓库页面


====================================================================
📋 工具界面说明
====================================================================

顶部：标题栏
  - 显示工具名称

项目信息区：
  - 显示仓库地址和用户名

快速操作区：
  - 6 个彩色按钮，点击即可操作

操作日志区：
  - 显示所有操作的详细日志
  - 带时间戳和状态图标

  - ✗ 错误（红色）
  - ⚠ 警告（橙色）
  - ℹ 信息（蓝色）

底部状态栏：
  - 显示当前操作状态


====================================================================
🎯 典型使用流程
====================================================================

首次使用：
---------
1. 双击 "启动工具.bat"
2. 点击 "一键上传到 GitHub"（输入密码）
3. 点击 "启用 GitHub Actions"（在浏览器中启用）
4. 点击 "触发自动构建"
5. 点击 "查看构建状态"（等待 10-20 分钟）
6. 下载构建完成的程序

创建发布版本：
-----------
1. 点击 "创建发布版本"
2. 输入版本号（如 v1.0.0）
3. 点击 "创建"
4. 等待 15-30 分钟自动构建
5. 访问 Releases 页面下载

日常更新代码：
-----------
1. 修改代码后
2. 点击 "一键上传到 GitHub"
3. 自动触发构建

⚠️ 常见问题
====================================================================

Q: 工具无法启动？
A: 确保已安装 Python 3.7+
   运行: python --version 检查

Q: 上传失败？
A: 检查：
   - 用户名和密码是否正确
   - 网络连接是否正常
   - Git 是否已安装

Q: 密码输入后无反应？
A: 这是正常的，Git 正在后台处理
   请耐心等待几秒钟

Q: 构建一直不开始？
A: 确保已启用 GitHub Actions
   点击 "启用 GitHub Actions" 按钮

Q: 想用 Token 代替密码？
A: 1. 访问 https://github.com/settings/tokens
   2. 生成新 token（勾选 repo 权限）
   3. 在密码框输入 token
